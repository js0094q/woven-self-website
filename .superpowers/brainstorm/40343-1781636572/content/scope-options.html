Which scope should the design spec own?

The repo currently has a live author/preorder page plus untracked campaign material. Pick the boundary before we define the design.

A. Website Copy Update

Update the live author page and metadata only. Fastest, least surface area, but leaves campaign assets inconsistent.

B. Campaign Asset System

Define the reusable visual/copy system for social, email, print, and exports. Strong for launch assets, but does not directly fix the live page.

C. Integrated Campaign + Website

One design system and message architecture, then apply it to the live author page and campaign templates. Recommended for consistency.
